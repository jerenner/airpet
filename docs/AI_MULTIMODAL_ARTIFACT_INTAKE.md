# AI Multimodal Artifact Intake + Extraction Contract (Checkpoint 2)

Schema version: `2026-03-14.multimodal-intake.checkpoint2`

Checkpoint 1 introduced deterministic PDF/image artifact intake.
Checkpoint 2 adds deterministic extraction schema validation (`regions`, `dimensions`, `symbols`) and machine-readable review-envelope primitives before any geometry write workflows.

## Checkpoint 1: Artifact Intake Endpoints

### `POST /api/ai/artifacts/upload`
Multipart upload endpoint for AI-planning artifacts.

Accepted file field names:
- `artifact` (preferred)
- `file` (alias)

Optional form fields:
- `source_path` (original local/source path for provenance checks)
- `source_label` (free-form source tag)

Accepted MIME classes:
- `application/pdf`
- `image/png`
- `image/jpeg`
- `image/webp`

Response shape:
```json
{
  "success": true,
  "schema_version": "2026-03-14.multimodal-intake.checkpoint1",
  "artifact": {
    "artifact_id": "artifact_...",
    "mime_type": "application/pdf",
    "size_bytes": 12345,
    "sha256": "...",
    "original_filename": "detector_sketch.pdf",
    "stored_filename": "artifact_..._detector_sketch.pdf",
    "created_at": "2026-03-14T12:34:56.789Z",
    "updated_at": "2026-03-14T12:34:56.789Z",
    "source_label": "user-dropbox",
    "source_path_input": "/path/origin.pdf",
    "source_path_resolved": "/path/origin.pdf",
    "source_path_exists": true,
    "source_path_is_file": true,
    "source_path_within_workspace": false
  }
}
```

### `GET|POST /api/ai/artifacts/list`
Deterministic artifact listing.

Inputs:
- `limit` (non-negative integer, default `50`)
- `include_missing_files` (boolean, default `false`)

Behavior:
- sorted by `(created_at DESC, artifact_id DESC)`
- when `include_missing_files=false`, metadata entries with missing blob files are filtered out

### `GET /api/ai/artifacts/<artifact_id>`
Fetch deterministic metadata for one artifact id.

## Checkpoint 2: Extraction Schema + Review Envelope

Implementation module: `src/ai_multimodal_extraction_schema.py`

### Normalized extraction payload shape
Generated by:
- `normalize_extraction_payload(payload, artifact_metadata=...)`

```json
{
  "schema_version": "2026-03-14.multimodal-intake.checkpoint2",
  "artifact_id": "artifact_...",
  "artifact_sha256": "...",
  "extraction_id": "extract_...",
  "regions": [
    {
      "region_id": "region_a",
      "label": "outer vessel",
      "page_index": 0,
      "bbox": {"x": 0.1, "y": 0.1, "width": 0.6, "height": 0.7},
      "confidence": 0.93,
      "provenance": {
        "artifact_id": "artifact_...",
        "artifact_sha256": "...",

    {

      "provenance": {
        "artifact_id": "artifact_...",

        "page_index": 0,
        "source": "ocr-text"
      }
    }
  ],
  "symbols": [
    {
      "symbol_id": "sym_a",
      "region_id": "region_a",
      "symbol_type": "annotation",
      "text": "beam axis",
      "confidence": 0.74,
      "provenance": {
        "artifact_id": "artifact_...",
        "artifact_sha256": "...",
        "page_index": 0,
        "source": "manual-markup"
      }
    }
  ],
  "stats": {
    "region_count": 1,
    "dimension_count": 1,
    "symbol_count": 1
  }
}
```

Deterministic schema guarantees:
- strict confidence range validation (`[0,1]`)
- deterministic ID ordering for regions/dimensions/symbols
- unique IDs per collection
- dimensions/symbols must reference known `region_id`
- provenance links must match canonical artifact metadata (`artifact_id`, `sha256`)
- deterministic `extraction_id` generation when omitted

Current review status enum:
- `pending_review`
- `approved`
- `needs_changes`
- `rejected`

## Persistence Layout

Per-session artifact store under project directory:
- `<projects_dir>/.airpet_ai_artifacts/manifest.json`
- `<projects_dir>/.airpet_ai_artifacts/blobs/*`

Manifest stores normalized metadata/provenance for reproducible downstream extraction/planning steps.
